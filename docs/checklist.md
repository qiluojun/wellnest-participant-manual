# Quick Checklist

Use this checklist when you are short on time or want to confirm that everything is on track.

## Before You Start

- [ ] Installed both apps: **WellNest** and **WellNest Health**. See [Install the apps](setup.md).
- [ ] Granted all permissions. See [Grant permissions](permissions.md).
- [ ] Confirmed that the red exclamation mark has disappeared. See [How to confirm permissions are complete](permissions.md#how-to-confirm-permissions-are-complete).
- [ ] Allowed notifications. See [Notification settings](permissions.md#notification-settings).
- [ ] Completed the Pre-Study Survey. See [Pre-Study Survey](surveys.md#pre-study-survey).
- [ ] Confirmed WellNest is running in the background. See [Keeping the app running in the background](settings.md#keeping-the-app-running-in-the-background).
- [ ] Uninstalled WellNest Health only after permission setup was complete. See [When can I uninstall WellNest Health?](permissions.md#when-can-i-uninstall-wellnest-health).

## Every Day

- [ ] Morning Survey submitted between **6:00-10:00 am**. The app will send a reminder. See [Daily Surveys](surveys.md#daily-surveys).
- [ ] Evening Survey submitted between **8:00 pm-12:00 am**. The app will send a reminder. See [Daily Surveys](surveys.md#daily-surveys).
- [ ] WellNest is still running in the background. See [Keeping the app running in the background](settings.md#keeping-the-app-running-in-the-background).

## Every Monday

- [ ] Weekly Survey completed. If missed, go to **Help screen** -> **Weekly Survey**. See [Weekly Survey](surveys.md#weekly-survey).

## If Something Looks Wrong

- [ ] Red exclamation mark on Permissions page -> re-check the missing item. See [Permissions](permissions.md).
- [ ] App was closed -> relaunch the app. See [Keeping the app running in the background](settings.md#keeping-the-app-running-in-the-background).
- [ ] Upload failed -> open the **Data Upload screen** and tap the upload button. See [Manual data upload](settings.md#manual-data-upload).
- [ ] Anything else -> email [study-support@mcl.iis.u-tokyo.ac.jp](mailto:study-support@mcl.iis.u-tokyo.ac.jp). See [I need help](troubleshooting.md#i-need-help).
